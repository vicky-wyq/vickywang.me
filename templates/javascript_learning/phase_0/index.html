Scope, Function, Variable | 2025.10-11




Scope

  • One big box that lives as long as the program runs.
  • Variables inside it (g, window, etc.) are always reachable.

Variable

  • When a function runs, JavaScript creates a temporary box for its variables (n, temp, etc.).
  • When the function finishes and no one else references that box, it’s destroyed.

Function

  • When a function returns another function, that inner one keeps a pointer to the outer box.
  • Because something still points to the box, JavaScript doesn’t throw it away.
  • That’s why count “stays alive”—the box (the scope) is still referenced.



Global: one shared box, persistent, public.

 
  let g = 0;
  function bump() { g++; return g; }

Local (no closure): new box each time, temporary.

 
  function bump() { let n = 0; n++; return n; }

Closure: private box tied to a specific returned function, persistent but hidden. 

 
  function makeCounter() {
    let count = 0;               // local box
    return function() {          // inner function keeps reference
      count++;
      return count;
    };
  }
  const c = makeCounter();


Learnings

  • new box each call
  • variables live in that box
  • closures keep boxes alive
  • global boxes never die unless nuke them.
