User research

Opportunity and use cases

Redditors are less welcoming of promoted content, topic and content are about honesty and transparency. People like to read and share Reddit thoughts before or after purchase.

My goal is to design a keywords analysis tool with sentiment score for Reddit. This app will help the business owner understand their customer.

The keywords analysis tool collects and analyzes large volumes of data from Reddit API. Displays data on the home page with filters based on the keywords set

Use cases can be

Accompany want to know how people feel about a new feature launched, or a new item released.

They also want to understand the positive and negative experiences with the brand and products. understand the reason based on a bunch of data

The customer support team is informed in real-time about any negative comments.

The keywords on Reddit get detected extremely quickly

Usability testing

Tasks

Users should be able to add and delete the keywords

Users should be able to see how popular the keywords in Reddit posts were and what was the sentiment behind them.

Users should be able to filter data by a certain date range or a single date

Users should be able to click the link to see the original Reddit posts. For example, short the table with negative sentiment or most popular posts, open the most popular link to see the details

Feedback from the testing

It is a bit ambiguous to navigate from keywords to the dashboard. each keyword has its own dashboard. Do I still need to have the dashboard tag on the left navigation panel? Or only have a list of keywords, move the setting to the top right.

When comparing data, the user has to open two windows to compare, maybe there is a better way to compare the data side by side
